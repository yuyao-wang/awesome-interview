## Study Notes

### [BV16d] Roadmap — 5-Stage LLM Learning Path (Overview)

This video series is built from real ByteDance/Meituan interview experience. It lays out a 5-stage roadmap for preparing for LLM engineer interviews:

| Stage | Topics |
|-------|--------|
| 1 | Mathematical Foundations & Python — linear algebra, probability, backprop |
| 2 | Deep Learning Core — CNNs, RNNs, optimization, batch norm |
| 3 | NLP & Transformers — attention, BERT, RoPE |
| 4 | Large Model Architecture — GPT variants, LLaMA, scaling laws |
| 5 | Fine-tuning & Alignment — LoRA, RLHF, PPO, DPO |

**Key insight from the author:** Interviewers at ByteDance don't just ask "what is X" — they dig into *why* design choices were made (e.g., why RoPE instead of absolute position encoding, why LayerNorm instead of BatchNorm in Transformers).

**Mindset for this prep:** Don't just memorize answers. For each topic, be able to explain:
1. What it is
2. Why it was designed this way
3. What tradeoffs it makes
4. How it compares to alternatives

### Pyre Code — Overview (68 Problems)

Problems grouped by difficulty/topic:
- **P1–10:** Basics — ReLU, Softmax, Linear, LayerNorm, Attention, MHA, BatchNorm, RMSNorm, Causal Attention, GQA
- **P11–20:** Advanced Attention — Sliding Window, Linear Attention, GPT2 block, KV Cache, MLP, Cross-entropy, Dropout, Embedding, GELU, Weight Init
- **P21–40:** Training — LoRA, DPO, PPO, GRPO, reward models, quantization (GPTQ, AWQ)
- **P41–60:** Systems — Flash Attention, PagedAttention, pipeline/tensor parallel
- **P61–68:** Frontier — MLA, MTP, MoE, MCTS, DiT, DDIM, Flow Matching

**Strategy:** Start from P1 (basics). Even if the concept seems simple (ReLU), the act of implementing it cleanly + explaining it in an interview is the actual skill being trained.
